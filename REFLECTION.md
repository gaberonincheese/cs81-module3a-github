What did you learn about using Git and GitHub?

I've used GitHub before, so I didn't necessarily learn much about it through this assignment, but I did learn about Git. I appreciated that I could access and edit my repos though a controlled environment available through one consolidated shell and hope to learn to use it more effectively with time. 

What was most confusing or frustrating?

I found the instructions on the assignment quite vague, which led to frustration. For example, after I cloned my repository, I couldn't find any indication where it had gone. I had to search my local disk to find it. I also found the usage of the word "touch" without a definition confusing, and had to look it up to understand its meaning. It was only after a lot of fiddling around with the shell that I felt I was able to get the hang of using the basic commands listed in the assignment. Hopefully my ignorance as to the proper terminology / commands will be rectified with more usage.

Why do you think developers use Git?

I can definitely understand why developers use Git. Rather than having to navigate multiple windows to push / pull from GitHub, they have access to everything through Git Bash's shell. That centralization must make the process much simpler.

How might this help with larger projects in the future?

I'll be able to more readily parse larger projects, as well as maintain my project through a singular, readily controllable place. I think this would take a lot of time to learn, though.
